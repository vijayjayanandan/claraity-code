# Subagent Demonstration Plan

## Summary
Demonstrate the three available subagents (code-reviewer, test-writer, doc-writer) by delegating sample tasks to each one and showing their specialized capabilities.

## Context
The system has a subagent architecture that allows delegating specialized tasks to focused agents with:
- Independent conversation context (no pollution)
- Specialized system prompts for domain expertise
- Configurable LLM models and tool access
- Isolated message stores for persistence

Available subagents:
1. **code-reviewer** - Reviews code for quality, security, performance
2. **test-writer** - Writes comprehensive test suites
3. **doc-writer** - Creates technical documentation

## Implementation Steps

### 1. Demonstrate code-reviewer subagent
- [ ] Create a sample Python function with potential issues
- [ ] Delegate to code-reviewer for analysis
- [ ] Show the review output (security, performance, quality feedback)

### 2. Demonstrate test-writer subagent
- [ ] Use the same sample function
- [ ] Delegate to test-writer to generate test cases
- [ ] Show the generated test suite with edge cases

### 3. Demonstrate doc-writer subagent
- [ ] Delegate documentation task for the sample function
- [ ] Show generated docstrings, usage examples, and API docs

### 4. Show subagent isolation
- [ ] Verify each subagent has independent context
- [ ] Show transcript files in `.clarity/sessions/subagents/`
- [ ] Demonstrate no context pollution between agents

## Verification
- [ ] All three subagents execute successfully
- [ ] Each produces specialized output appropriate to their role
- [ ] Transcript files are created for each subagent
- [ ] No errors or context pollution

## Trade-offs
**Pros:**
- Clean separation of concerns
- Specialized expertise per domain
- Independent context prevents pollution
- Can use different LLM models per subagent

**Cons:**
- Slight overhead from spawning separate contexts
- Each subagent doesn't see main conversation history

## Alternatives Considered
- **Single agent with role-switching**: Would pollute context and lose specialization
- **Manual prompting**: Less structured, harder to maintain consistency
