### The 4 Code Review Issues to Fix Before Compiling
Based on the code review that was done:
1. **`enrichmentError` silently discards the error** ‚Äî `App.tsx:112-114` just does `CLEAR_ENRICHED_PREVIEW` without showing the user any error message. Fix: also dispatch `ERROR` action so the user sees what went wrong.
2. **`sidebar-provider.ts` doesn't forward `history`** ‚Äî line 446 sends `{ type: 'enrich_prompt', content: msg.content }` without `msg.history`. Fix: include `history: msg.history` in the send.
3. **`enrichedPreview` condition in InputBox** ‚Äî the enrichment preview textarea shows `{enrichedPreview && !enrichmentLoading && ...}` which means streaming deltas (first delta clears `enrichmentLoading`) appear correctly, but the loading spinner disappears on first delta. This is actually correct per `ENRICHMENT_DELTA` reducer. Keep as-is.
4. **`handleSend` doesn't send the enriched preview when user presses Enter** ‚Äî when `enrichedPreview` is set and user presses Enter on the main textarea (not the Send button in the preview), it falls through and calls `onRequestEnrichment` again if not handled correctly. Check the condition: `if (enrichmentEnabled && !enrichedPreview && !enrichmentLoading)` ‚Äî this correctly skips re-enriching when preview exists and goes to `onSend`. But `onSend` uses `trimmed` (the original draft text), not the enriched preview. Fix: when `enrichedPreview` is set and the user presses Enter (or the send button), use `editedPreview ?? enrichedPreview` as the content.

Let me re-check issue 4 in the code:
- `handleSend` (line 160-177): if `enrichedPreview` exists, it falls through to `onSend(trimmed, ...)` using the ORIGINAL text, not the enriched text. This is a bug: pressing Enter when preview is showing sends the original draft, not the enriched text. Fix: when `enrichedPreview` is truthy, call `handleSendEnriched()` instead.
