# Plan: Centralize LLM Provider Error Classification

## Goal
Move provider error classification out of `agent.py`'s streaming exception handler and into
`failure_handler.py` so there is one tested, maintainable source of truth. Users see actionable
messages (e.g. "Model 'gpt-5' does not exist") instead of generic "provider error".

## Constraints
- `handle_api_error()` public tuple contract `(bool, str)` MUST NOT change -- retry helpers and
  existing tests all depend on it.
- Error store recording, structured logging, pause flow in `agent.py` MUST be preserved unchanged.
- `ErrorEvent` schema unchanged -- no TypeScript changes needed (`error_type` is free-form `str`).
- No emojis in Python source (Windows cp1252 invariant).

---

## Slice 1 -- Enrich exception classes in `failure_handler.py`

**File:** `src/llm/failure_handler.py`

Add `user_message: str`, `event_type: str`, `retryable: bool` to `LLMError` base class as
constructor args with defaults. Update each subclass to set sensible defaults:

| Class | event_type | retryable | default user_message |
|---|---|---|---|
| `LLMError` (base) | `"provider_error"` | `True` | `"Provider error"` |
| `InvalidAPIKeyError` | `"config_error"` | `False` | `"API key is invalid or missing. Check Settings."` |
| `InvalidModelError` | `"config_error"` | `False` | `"Model not found. Check the model name in Settings."` |
| `ContentPolicyViolationError` | `"content_policy"` | `False` | `"Request blocked by provider content policy."` |
| `ContextLengthExceededError` | `"config_error"` | `False` | `"Context length exceeded."` |
| `RateLimitError` | `"rate_limit"` | `True` | `"Rate limit hit. Retrying..."` |
| `TimeoutError` | `"provider_timeout"` | `True` | `"Request timed out."` |

New subclass: `InvalidRequestError(LLMError)` -- catches bad param values (temperature, max_tokens
etc.) that don't fit the above buckets. `event_type="config_error"`, `retryable=False`.

New subclass: `NetworkError(LLMError)` -- for connection/network failures. `event_type="network"`,
`retryable=True`.

**Backward compat:** existing code that does `raise InvalidAPIKeyError()` with no args still works
because all new args have defaults.

## Slice 3 -- Add `classify_provider_error()` in `failure_handler.py`

**File:** `src/llm/failure_handler.py`

New module-level function (not a method):

```python
def classify_provider_error(exc: Exception) -> LLMError:
    """Classify any provider exception into a typed LLMError subclass.
    
    Returns a typed LLMError with user_message, event_type, and retryable set.
    Never raises -- always returns an LLMError even for unknown exceptions.
    """
```

Classification order (first match wins):

1. **Timeout** -- `"timeout"` / `"timed out"` in type name or message -> `TimeoutError`
2. **Rate limit** -- `"rate_limit"` / `"rate limit"` / `"too many requests"` / `"429"` -> `RateLimitError`
3. **Network** -- `"connection"` / `"network"` / `"503"` / `"502"` / `"504"` / `"service unavailable"` -> `NetworkError`
4. **Invalid API key** -- `"invalid_api_key"` / `"invalid api key"` / `"incorrect api key"` / `"401"` / `"authentication_error"` -> `InvalidAPIKeyError`
   - NOTE: bare `"authentication"` and `"unauthorized"` REMOVED -- too broad
5. **Content policy** -- `"content_policy_violation"` / `"content policy"` -> `ContentPolicyViolationError`
6. **Invalid model** -- `"invalid_model"` / `"invalid model"` / `"model not found"` / `"404"` -> `InvalidModelError`
7. **Invalid request / bad params** -- `"invalid_request_error"` / `"invalid value for"` / `"must be"` phrases -> `InvalidRequestError`
   - NOTE: bare `"invalid_request"` REMOVED -- too broad, was catching content policy
8. **Context length** -- `"context_length_exceeded"` / `"maximum context length"` -> `ContextLengthExceededError`
9. **Unknown** -- `NetworkError` with retryable=True (conservative, preserves existing behaviour)

The `user_message` for config errors extracts the provider's message from the sanitized root cause
string using `_sanitize_error_message()`, prefixed with `"API configuration error: "`. Content
policy and timeout get fixed strings (no provider detail needed).

## Slice 4 -- Make `handle_api_error()` delegate to classifier

**File:** `src/llm/failure_handler.py`

Replace the existing keyword-loop body of `handle_api_error()` with:

```python
def handle_api_error(self, error: Exception) -> tuple[bool, str]:
    classified = classify_provider_error(error)
    return classified.retryable, classified.user_message
```

This is a pure drop-in. All callers (`execute_with_retry`, `execute_with_retry_async`,
`handle_rate_limit`, `handle_timeout`) get the same tuple they always got. All existing tests
pass unchanged -- the assertions check `is_retryable` bool and presence of words like "fatal"
in the message. We need to verify "fatal" still appears or update the handful of tests that
check for it (the new messages won't say "Fatal error (ExceptionType):" -- we'll update those
tests to check the actual meaningful content instead).

Also update retry helpers to re-raise the classified subtype instead of wrapping in plain
`LLMError` -- so when a typed exception propagates up to `agent.py`, it still carries its
metadata:

```python
# In execute_with_retry fatal branch (currently line 360):
classified = classify_provider_error(e)
raise classified from e   # preserves type + metadata
```
