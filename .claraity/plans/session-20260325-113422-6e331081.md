# Plan: Replace task_* tools with claraity_* as sole task management system

## Summary
Remove the old `task_create/update/list/get` tools and `TaskState` entirely.
Wire the BeadStore (`claraity_*` tools) to the TUI TodoBar so it shows session-active beads.
The agent uses only `claraity_*` tools for all task tracking going forward.

## Context

### Old system (to remove)
- `src/tools/task_state.py` - TaskState class (in-memory + JSON per session)
- `src/tools/planning_tool.py` - TaskCreateTool, TaskUpdateTool, TaskListTool, TaskGetTool
- Registered in `agent.py` lines 791-794
- Drives TodoBar via `notify_todos_updated(self.task_state.get_todos_list())`
- Side-effect trigger at `agent.py:2514`: `if tool_name in ("task_create", "task_update")`
- Error-budget tracking uses `task_state.error_budget_resume_count`, `.successful_tools_since_resume`, `.last_stop_reason`, `.current_task_id`
- `todo_state` property (lines 175-182) feeds context_builder and hydrator
- Session persistence: `todos.json` per session in `.clarity/sessions/{id}/todos.json`
- Hydrator (`hydrator.py`) restores todos from JSONL message metadata

### New system (to wire up)
- `src/claraity/claraity_beads.py` - BeadStore (SQLite, cross-session)
- `src/tools/claraity_tools.py` - BeadCreateTool, BeadUpdateTool, BeadReadyTool, BeadBlockTool
- Registered in `agent.py` lines 854-857 (already done)
- Bead statuses: `open`, `in_progress`, `closed`
- TodoBar expects dicts with keys: `status`, `subject`/`content`, `activeForm`
- BeadStore has no `subject` or `activeForm` fields — uses `title`

### TodoBar field mapping
BeadStore -> TodoBar:
- `title` -> `subject` (and `activeForm` when in_progress)
- `status`: `open` -> `pending`, `in_progress` -> `in_progress`, `closed` -> `completed`

### Error-budget counters on TaskState
`error_budget_resume_count`, `successful_tools_since_resume` are internal agent counters
NOT related to task content. They must be preserved but can live directly on the agent,
not in TaskState. TaskState will be removed, so these move to simple agent attributes.

### `last_stop_reason` and `current_task_id`
Also internal agent state. Move to direct attributes on CodingAgent.
`get_pending_summary()` needs a replacement that reads from BeadStore.

## Trade-offs
- **TodoBar now shows ALL non-closed beads** (cross-session tasks could appear). This is
  acceptable: the bar accurately reflects "what's in play" even across sessions.
- **Session hydration drops todo restore** - beads are already in SQLite so hydration
  from JSONL is not needed. No data loss.
- **Error budget counters move to plain agent attrs** - simpler, no persistence needed
  (they reset each session anyway).
