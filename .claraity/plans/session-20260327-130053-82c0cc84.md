# Disconnect/Reconnect + Disabled Input When Disconnected

## Summary
Add a disconnect/reconnect button to the BottomBar and disable the chat input
(textarea + send button) when the server is disconnected.

## Files to Change (5)

1. `claraity-vscode/src/types.ts` — add `disconnectServer` and `reconnectServer` WebViewMessage variants
2. `claraity-vscode/src/sidebar-provider.ts` — handle the two new messages, call connection.disconnect() / connection.restart()
3. `claraity-vscode/webview-ui/src/components/BottomBar.tsx` — add disconnect/reconnect button, wire up props
4. `claraity-vscode/webview-ui/src/components/InputBox.tsx` — accept `disabled` prop, apply to textarea + send button
5. `claraity-vscode/webview-ui/src/App.tsx` — pass `onDisconnect`/`onReconnect` to BottomBar, pass `disabled={!state.connected}` to InputBox

## Design Decisions

### Button placement
The disconnect/reconnect button goes in `BottomBar` next to the "Connected/Disconnected"
status text -- it's the most natural location since the status is already there.

### Single button, two states
- When connected: shows a plug-with-slash icon (`codicon-debug-disconnect`) with tooltip "Disconnect"
- When disconnected: shows a plug icon (`codicon-plug`) with tooltip "Reconnect"
One button, two states -- avoids cluttering the bar.

### Input disabled scope
Disable the `<textarea>` and the send `<button>` only. The interrupt button is
irrelevant when disconnected (no stream running), and the attach/enrichment buttons
are harmless to leave enabled (they only affect local state). Keep it minimal.

### No new WebViewMessage types needed for InputBox
`InputBox` already receives `isStreaming` prop. We add a `connected` prop alongside it.
The send button is already conditionally styled/labelled -- we extend the same pattern.

## Implementation Steps

### 1. `claraity-vscode/src/types.ts`
Add to `WebViewMessage` union:
```ts
| { type: 'disconnectServer' }
| { type: 'reconnectServer' }
```

### 2. `claraity-vscode/src/sidebar-provider.ts`
Add two cases to the `switch (msg.type)` block:
```ts
case 'disconnectServer':
    this.connection?.disconnect();
    break;
case 'reconnectServer':
    this.connection?.restart();
    break;
```

### 3. `claraity-vscode/webview-ui/src/components/BottomBar.tsx`
- Add `onDisconnect: () => void` and `onReconnect: () => void` to `BottomBarProps`
- Add button in `bottom-left` (after the status text):
  - When `connected`: `codicon-debug-disconnect`, tooltip "Disconnect", calls `onDisconnect`
  - When `!connected`: `codicon-plug`, tooltip "Reconnect", calls `onReconnect`

### 4. `claraity-vscode/webview-ui/src/components/InputBox.tsx`
- Add `connected: boolean` to `InputBoxProps`
- On `<textarea>`: add `disabled={!connected && !isStreaming}`
- On send `<button>`: already has `disabled` logic implied by streaming -- add `disabled={!connected}` when not streaming
- Add `placeholder="Disconnected from server..."` when `!connected`

### 5. `claraity-vscode/webview-ui/src/App.tsx`
- Pass to `BottomBar`:
  ```tsx
  onDisconnect={() => postMessage({ type: 'disconnectServer' })}
  onReconnect={() => postMessage({ type: 'reconnectServer' })}
  ```
- Pass to `InputBox`:
  ```tsx
  connected={state.connected}
  ```
