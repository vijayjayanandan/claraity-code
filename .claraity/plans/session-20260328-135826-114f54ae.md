# Compaction UX -- Indicator + Summary Rendering

## Summary
Fix two gaps: (1) show "Compacting conversation..." in StreamingStatus during compaction;
(2) render the compaction summary as a collapsible system card in chat (not invisible).

## Context

### The root cause (three gates)
- **Gate 1** -- `reducer.ts:329` drops all MESSAGE_ADDED with role user/assistant/system → summary never enters state
- **Gate 2** -- `serializers.py` message_added payload omits `is_compact_summary` → webview can't distinguish it
- **Gate 3** -- `dispatch.ts:307-308` has `context_compacting`/`context_compacted` as no-op breaks → no status update

### Key existing patterns
- `StreamingStatus` already shows "Thinking...", tool names, "Streaming..." -- we add "Compacting conversation..." as highest-priority case
- `ThinkingBlock` uses `<details open={...}>` pattern -- compaction summary card will follow the same collapsible pattern
- `TimelineEntry` is a discriminated union -- we add `compaction_summary` variant
- `isStreaming` stays `true` during compaction (compaction is mid-stream) -- `StreamingStatus` is already visible

### Wire protocol
- `context_compacting` arrives as `{type: "context_compacting", tokens_before: N}`
- `context_compacted` arrives as `{type: "context_compacted", messages_removed: N, tokens_before: N, tokens_after: N}`
- summary arrives as `store/message_added` with `role: "user"`, content starting with `[Conversation summary...]`
- We add `is_compact_summary: true` to the message_added payload in serializers.py

## Implementation Steps

### Slice 1 -- Python: forward is_compact_summary in wire protocol
File: `src/server/serializers.py`

In `serialize_store_notification()`, in the `StoreEvent.MESSAGE_ADDED` branch, at the final
`return` (line 271), add `is_compact_summary` to the data dict when set on the message meta:

```python
return {
    "type": "store",
    "event": "message_added",
    "data": {
        "uuid": msg.uuid,
        "role": msg.role,
        "content": _content_to_str(msg.content),
        "stream_id": getattr(msg.meta, "stream_id", None),
        **({"is_compact_summary": True} if getattr(msg.meta, "is_compact_summary", False) else {}),
    },
}
```

Also need to suppress the compact_boundary system message (role="system", event_type="compact_boundary")
from being sent at all -- it's internal bookkeeping, not useful to the webview. Add a check before
the final return:

```python
# Suppress internal bookkeeping messages
if msg.is_system and msg.meta and msg.meta.event_type == "compact_boundary":
    return None
```

## Trade-offs
- We strip the `[Conversation summary - earlier messages were compacted to free context space]` prefix
  in the card header label -- it's redundant with the card title. The content shown is the actual summary.
- `isCompacting` does NOT affect `isStreaming` -- both can be true simultaneously. StreamingStatus
  is already visible (isStreaming=true), we just override the status text.
- compact_boundary messages are suppressed at the serializer level (return None) rather than
  the reducer -- cleaner, avoids sending useless data over TCP.
