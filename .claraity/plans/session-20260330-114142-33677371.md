# Plan: Show Default Enrichment Prompt in Config Panel

## Summary
Extract the enrichment system prompt to `src/prompts/enrichment.py`, expose it via
`config_loaded`, and update the ConfigPanel to show it as a placeholder and provide
a "Reset to default" button.

## Files Affected
- `src/prompts/enrichment.py` — NEW: holds `ENRICHMENT_SYSTEM_PROMPT` constant
- `src/server/stdio_server.py` — remove class constant, import from prompts
- `src/server/config_handler.py` — add `default_system_prompt` to config_loaded payload
- `claraity-vscode/webview-ui/src/components/ConfigPanel.tsx` — read default, show as placeholder, add Reset button, rename label

---

## Implementation Steps

### 1. Create `src/prompts/enrichment.py`
New file with the constant extracted verbatim from `stdio_server.py`:

```python
"""Default prompts for the prompt enrichment feature."""

ENRICHMENT_SYSTEM_PROMPT = (
    "You are an expert prompt engineer for a coding agent called ClarAIty. "
    "Your job is to rewrite a user's short or vague coding request into a clear, "
    "precise instruction that captures exactly what they want done and why.\n\n"
    "Rules:\n"
    "- Output plain prose only — no bullet points, no section headers, no markdown.\n"
    "- 1 to 3 sentences maximum.\n"
    "- Preserve the user's intent exactly — do not add requirements they did not ask for.\n"
    "- If the request is already clear, return it as-is."
)
```

### 2. Update `src/server/stdio_server.py`
- Remove `_ENRICHMENT_SYSTEM_PROMPT` class attribute (~line 2084)
- Add import at top of `_handle_enrich_prompt` (or module level):
  `from src.prompts.enrichment import ENRICHMENT_SYSTEM_PROMPT`
- Replace `self._ENRICHMENT_SYSTEM_PROMPT` with `ENRICHMENT_SYSTEM_PROMPT`

### 3. Update `src/server/config_handler.py`
In `get_config_response()`, add `default_system_prompt` to the prompt_enrichment dict:

```python
from src.prompts.enrichment import ENRICHMENT_SYSTEM_PROMPT

config_dict["prompt_enrichment"] = {
    "model": pe.model,
    "system_prompt": pe.system_prompt,
    "default_system_prompt": ENRICHMENT_SYSTEM_PROMPT,
}
```

### 4. Update `ConfigPanel.tsx`

**a) `parseSnapshot()`** — read `default_system_prompt` from the server payload and
add `enrichmentDefaultPrompt: string` to `ConfigSnapshot`:

```ts
enrichmentDefaultPrompt: typeof (cfg.prompt_enrichment as Record<string, unknown>)?.default_system_prompt === "string"
  ? (cfg.prompt_enrichment as Record<string, unknown>).default_system_prompt as string
  : "",
```

**b) State** — add `const [enrichmentDefaultPrompt, setEnrichmentDefaultPrompt] = useState("")`

**c) `applySnapshot()`** — add `setEnrichmentDefaultPrompt(snap.enrichmentDefaultPrompt)`

**d) `snapshotEqual()`** — no change needed (default is read-only, not user-editable)

**e) `currentSnapshot` useMemo** — no change needed (default not part of dirty tracking)

**f) Textarea** — change placeholder from `"(use built-in default)"` to `enrichmentDefaultPrompt`

**g) Label** — rename `"System Prompt"` to `"Enrichment Prompt"`

**h) Reset button** — add inline next to the label using the `Field` label prop (which accepts `React.ReactNode`):

```tsx
<Field label={
  <span style={{ display: "flex", alignItems: "center", gap: 6 }}>
    Enrichment Prompt
    {enrichmentSystemPrompt && (
      <button
        className="model-clear-btn"
        title="Reset to default"
        onClick={() => setEnrichmentSystemPrompt("")}
        tabIndex={-1}
      >
        Reset
      </button>
    )}
  </span>
}>
```

The Reset button only appears when the user has typed a custom value (i.e., `enrichmentSystemPrompt` is non-empty). Clicking it sets the value to `""`, which:
- Clears the textarea (showing the default as placeholder again)
- Is the correct semantic — empty string = use built-in default in the Python server

- `ConfigSnapshot` gains a read-only field that isn't part of dirty tracking — slightly asymmetric but correct since the default is server-controlled, not user-editable.
