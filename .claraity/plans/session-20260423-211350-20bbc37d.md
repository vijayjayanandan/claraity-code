## Bug 1: list_directory / glob / grep silently fail on dot-folders (.claraity)

### Root Causes

**Cause A: `_should_skip()` in GrepTool (~line 474) and GlobTool (~line 806)**
Both have a blanket guard that skips any file whose parent path contains a dot-prefixed component:
```python
if any(part.startswith(".") for part in file_path.parts[:-1]):
    return True
```
`.claraity/readme-drafts/foo.md` -> always skipped.

**Cause B: `filter_paths()` in ListDirectoryTool (~line 619)**
Calls `filter_paths()` which applies `.gitignore`. `.gitignore` has `.claraity/*` -> contents silently filtered.

### Fix A -- both `_should_skip` methods

Remove the blanket dot-prefix check. Expand `skip_dirs` to cover common noisy/cache dirs
that would otherwise be newly exposed:

```python
skip_dirs = {
    "node_modules", "__pycache__", ".git", ".venv", "venv",
    "dist", "build",
    # Web build/framework tool dirs
    ".next", ".nuxt", ".svelte-kit", ".turbo", ".yarn",
    # Editor/IDE dirs
    ".idea", ".vscode",
    # Python/typing cache dirs
    ".pytest_cache", ".mypy_cache", ".ruff_cache",
}
if any(part in skip_dirs for part in file_path.parts):
    return True
```

Generic dot-dirs not in this list (`.claraity`, `.hidden`, etc.) will now be searchable.
`filter_paths()` still applies `.gitignore`/`.claraityignore` AFTER `_should_skip` during
broad recursive searches, providing a second layer of filtering.

### Fix B -- ListDirectoryTool

Replace `filter_paths(all_entries)` with per-entry `is_blocked()` (claraityignore only).
`is_blocked` is already imported in `file_operations.py`. Remove the `all_entries` /
`allowed_entries` / `allowed_set` pattern; fold filtering into the single iteration:

```python
entries = []
# Only .claraityignore policy applies -- gitignore intentionally not used here.
# The user explicitly named this directory; gitignore should not censor its contents.
for entry in path.iterdir():
    blocked, _ = is_blocked(entry)
    if blocked:
        continue
    stat = entry.stat()
    ...
```

## Bug 2: grep files_with_matches looks like "no results"

### Root Cause
`_search_file()` in `FILES_WITH_MATCHES` mode returns `["match"]` -- a 1-item sentinel.
No count info is available. Bare file paths in output look like "nothing found" to the agent.

### Fix -- use COUNT internally for FILES_WITH_MATCHES

In the per-file loop, call `_search_file()` with `OutputMode.COUNT` when the effective mode
is `FILES_WITH_MATCHES`. COUNT mode returns `list(matched_lines)` (set of 0-indexed line
numbers with matches), so `len(matches)` gives real count. Format output as:
```
src/foo.py (2 matches)
src/bar.py (1 match)
```

**Multiline note (from planner review):** In multiline mode, one regex match can span
multiple lines. `matched_lines` covers all spanned lines, so `len()` gives matched *lines*,
not regex occurrences. This is acceptable -- the label is still unambiguous compared to bare
paths. No special-casing needed; the count is always at least 1 when a match exists.

**metadata["matches"] semantics:** Currently equals "number of matching files" in
FILES_WITH_MATCHES mode (it's `len(all_matches)` where each entry is one file path).
After the fix it remains "number of matching files" -- unchanged. `count` per file is
only used for the output string, not for the metadata total.

**head_limit / offset:** Applied after the loop to `all_matches` (list of formatted strings,
one per file) -- unaffected. Pagination is still at the file level.

```python
def test_shows_gitignore_hidden_entries(self, tmp_path, monkeypatch):
    """list_directory ignores .gitignore -- user explicitly chose this path."""
    monkeypatch.chdir(tmp_path)
    (tmp_path / ".gitignore").write_text("secret/\n")
    (tmp_path / "secret").mkdir()
    (tmp_path / "secret" / "data.txt").write_text("hello")
    tool = ListDirectoryTool()
    result = tool.execute(directory_path=str(tmp_path / "secret"))
    assert result.status == ToolStatus.SUCCESS
    assert "data.txt" in result.output

def test_hides_claraityignore_blocked_entries(self, tmp_path, monkeypatch):
    """.claraityignore-blocked entries are omitted from list_directory."""
    monkeypatch.chdir(tmp_path)
    (tmp_path / ".claraityignore").write_text("secret.txt\n")
    (tmp_path / "secret.txt").write_text("blocked")
    (tmp_path / "visible.txt").write_text("allowed")
    tool = ListDirectoryTool()
    result = tool.execute(directory_path=str(tmp_path))
    assert result.status == ToolStatus.SUCCESS
    assert "visible.txt" in result.output
    assert "secret.txt" not in result.output
```
